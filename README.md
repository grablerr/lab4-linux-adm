# Lab 4 DevOps

Лабораторная работа № 4. DevOps-автоматизация: systemd, bash-скрипты и Ansible

## Состав проекта

- `service.sh` — скрипт для запуска HTTP-сервиса
- `lab4-service.service` — unit-файл systemd для управления сервисом
- `lab4-healthcheck.sh` — скрипт для проверки доступности сервиса
- `inventory.example.ini` — пример inventory-файла для Ansible
- `site.yml` — playbook Ansible для развертывания сервиса

## Проверка работоспособности

Сервис предоставляет HTTP-страницу и доступен на порту 8000.

## Примечание

Файл `inventory.ini`, содержащий реальные данные (ключи, пароли и т.п.), не должен добавляться в репозиторий.
